png이미지와 svg이미지의 차이점

  • png 이미지

    모바일에서 선명하게 보이지 않음. 만약 png로 저장한 아이콘을 모바일에서 선명하게 보이게 하려면 2배 이상 사이즈로 저장해야함

  • svg 이미지

    아이콘이나 로고 같은 일러스트에서 제작한 이미지 저장가능 (사진 불가) 모바일에서도 선명하게 보이고 확대해도 깨지지 않음.

svg를 이미지태그로 코딩한 스타일

아이콘을 이미지태그로 코딩하면 마우스를 오버했을때 이미지 교체해야 하는 디자인일 경우 구현이 불가능하다.(진짜 안되는건 아닌데.....)
이미지의 위치를 디테일하게 조정하는게 어려움.

구글아이콘, 폰트어썸(Font Awesome) 등에서 제공하는 아이콘을 사용하는 방식

폰트어썸은 굉장히 많이 사용되었었음 (유료화되어서 지금 안씀)
SI업체 솔루션 개발이나 어플리케이션 개발에서는 많이 활용됨(홈페이지는 현재 인기가 없음)
아이콘 폰트를 만들어서 코드를 입력하면 아이콘이 나타는 방식
가벼움... 용량이 작아서 많이 쓰임

  • 1. 구글아이콘 홈페이지에서 아이콘 셋팅하고 (두께나 스타일) 아이콘 아무거나 클릭 (오른쪽에 코드가 나옴)
  • 2. 오른쪽 패널에서 Static icon font 하단에 있는 css 링크 주소를 복사해서 넣어줌 (단 한번만 넣어주면 됨)
  • 3. 실제 홈페이지에서 사용할 아이콘을 찾는다.
  • 4. 그 아이콘의 Inserting the icon 하단의 코드를 복사합니다.
  • 5. 그걸 아이콘이 들어갈 위치에 붙여넣음
  • 6. span태그는 i태그로 변경. 절대 class명 수정 X, 영문도 함부로 바꾸면 안됨. 그걸 바꾸면 아이콘이 바뀜
  • 7. css에서 크기와 색상 변경 가능
